# Setup

There are two ways to set up the Virtual Cortex. Pick one and follow it top to bottom, they reach the same end state.

| | [AI-driven setup](01a-setup-ai.md) | [Manual setup](01b-setup-manual.md) |
|---|---|---|
| **Time** | ~10 min | ~15 min |
| **You need** | An AI agent (Claude Code / Cursor / Claude Desktop), `uv`, and the mcp-logseq server | Just Logseq and a text editor |
| **What happens** | The agent scans your graph and builds the hubs and templates for you after you approve a plan | You drop ready-made files into your graph and paste one config block |
| **Bonus** | Detects your existing tags and merges them into the hub queries | Nothing to install, fully offline |
| **Best for** | People who already use an AI agent and want it done in one command | People who want zero dependencies, or do not use an AI agent |

## Which should I pick?

- **Use [AI-driven setup](01a-setup-ai.md)** if you already work with Claude Code, Cursor, or Claude Desktop. It is faster, it adapts to whatever is already in your graph, and the same skills then power daily capture and reviews.
- **Use [manual setup](01b-setup-manual.md)** if you do not use an AI agent, do not want to install anything, or just want to see exactly what lands in your graph.

You can always start manual and add the AI skills later, the manual setup is fully functional on its own.

## Prerequisites (both paths)

- Logseq installed (desktop version)
- A graph you are willing to add pages to (brand new or existing, this is purely additive)
